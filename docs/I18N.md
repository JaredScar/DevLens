# Internationalization (i18n)

## Current state

- `document.documentElement.lang` follows **Settings → General → Language** (BCP 47 string, default `en`).
- **Settings → Appearance → RTL layout** sets `dir="rtl"` on the document root for right-to-left scripts.

## Future: translation files

For full UI translation, the planned approach is **`@ngx-translate/core`** (or Angular built-in i18n with build-time locale bundles):

1. Add `@ngx-translate/core` and `@ngx-translate/http-loader` to the `app` workspace.
2. Provide `TranslateModule` / `provideTranslateService` in `app.config.ts`.
3. Move user-visible strings into `public/assets/i18n/<locale>.json` incrementally.
4. Use `translate` pipe or `TranslateService` in templates.

Extracting every string is a large follow-up; the shell above keeps locale and direction wired without blocking English-first development.
